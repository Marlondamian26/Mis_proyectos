Puedes ejecutar las pruebas de seguridad en cualquier momento usando el comando:
```powershell
.\scripts\run-security-tests.ps1 -targetUrl "http://localhost:3000"
```

Hay algunos puntos importantes a tener en cuenta para asegurar que las pruebas se ejecuten correctamente en el futuro:
1. **Requisitos Previos**:
   - Asegúrate de que OWASP ZAP esté instalado en `C:\Program Files\ZAP\Zed Attack Proxy\`
   - Verifica que Java esté instalado y configurado correctamente
   - La aplicación debe estar corriendo en `http://localhost:3000` cuando ejecutes las pruebas

2. **Permisos**:
   - Ejecuta PowerShell como administrador
   - Asegúrate de que el directorio `reports` exista en tu proyecto

3. **Consideraciones**:
   - Si actualizas la versión de OWASP ZAP, podrías necesitar actualizar el script
   - Si cambias el puerto de tu aplicación, recuerda actualizar el parámetro `-targetUrl`
   - Los reportes se guardarán en `.\reports\security-report.html`, sobrescribiendo el reporte anterior

4. **Solución de Problemas**:
   Si encuentras algún error en el futuro, verifica:
   - Que ZAP esté correctamente instalado
   - Que Java esté instalado y configurado
   - Que la aplicación esté corriendo
   - Que tengas permisos de administrador
   - Que el directorio `reports` exista
